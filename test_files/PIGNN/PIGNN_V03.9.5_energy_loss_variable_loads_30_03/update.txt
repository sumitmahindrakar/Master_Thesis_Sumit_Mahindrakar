Every displacement depends on every force — this is a global operation. With only 10 hops, nodes at the ends of the frame are blind to forces on the other end.

Why Fx=[1,1] worked: All forces identical → no information to propagate → local is enough.

Why Fx∈[-10,10] fails: Every node has a different force → model MUST propagate force info globally → 10 layers aren't enough.

The Fix: Global Pooling
Instead of adding more MP layers (expensive, diminishing returns, harder to optimize), give every node direct access to the global graph state.

Architecture change
text

CURRENT:  Input → Encoder → [10 MP layers] → Decoder → u

PROPOSED: Input → Encoder → [10 MP layers] → Global Pool → Concat → Decoder → u
After message passing, we:

Pool all node features → single graph-level vector
Broadcast it back to every node
Concatenate local + global features
Decode from the enriched representation
This gives every node "long-range vision" without adding MP layers.

Why it works physically
For a linear structure, displacement decomposes into:

Global mode (overall deformation pattern) — determined by the total load pattern → captured by global pool
Local detail (node-specific response) — determined by nearby forces → captured by MP layers
