node input feature are raw coord not normalized
#-----------------------------------------------------------------------------

The node encoder receives mixed-scale inputs. Coordinates dominate (0–30) while loads are squashed to [0,1]. This causes:

The network heavily biases toward coordinate features
Load information is practically invisible to the first layer
Training is unstable because gradients have wildly different magnitudes
#-----------------------------------------------------------------------------
what autograd actually computes is:

Python

grad_ux = autograd.grad(ux.sum(), coords)[0]
# grad_ux[j, 0] = ∂(Σᵢ ux_i) / ∂(x_j) = Σᵢ ∂ux_i/∂x_j

Due to message passing, ux_i depends on coordinates of ALL nodes, not just node i. So:

text

grad_ux[j, 0] = ∂ux_j/∂x_j          ← local (what you want)
              + Σ_{i≠j} ∂ux_i/∂x_j   ← cross-terms from MP (noise!)

This is NOT the local strain. The cross-terms contaminate everything. And it gets exponentially worse for higher derivatives
